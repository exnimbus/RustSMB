## Status

RustSMB is an SMB2/3 file-server core with virtual filesystem backends, direct TCP
transport, and SMB 3.1.1 over QUIC. It targets modern authenticated SMB
workloads with signing, encryption, compression, leases/oplocks, durable
handles, compound requests, WAN-friendly transfer sizing, and external-client
interop tests.

SMB1, NetBIOS name service/session-service discovery, host filesystem
passthrough, persistent durable handles, and Kerberos are currently out of scope.
Multi-channel and QUIC multipath are in scope but not fully implemented.

As of June 2026, you should consider this a pre-alpha release.

## Install

```sh
cargo add smb-server
```

Or in `Cargo.toml`:

```toml
[dependencies]
smb-server = "0.4"
```

## Run the example

```sh
cargo run -p minimal-smb-example
```

Listens on `0.0.0.0:4445`. Override with `SMB_LISTEN` and `SMB_ROOT`.

Shares: `public` (anon rw), `media` (anon ro), `home` (alice rw, bob ro).

## Embed

```rust
use smb_server::{Access, LocalFsBackend, Share, SmbServer};

let server = SmbServer::builder()
    .listen("0.0.0.0:4445".parse()?)
    .user("alice", "password")
    .share(
        Share::new("home", LocalFsBackend::new("/srv/home")?)
            .user("alice", Access::ReadWrite),
    )
    .build()?;

server.bind().await?;
server.serve().await?;
```

## Runtime config

Create a `ConfigHandle` before `serve()` and keep it in the task that owns admin/config updates:

```rust
use smb_server::{Access, LocalFsBackend, Share, SmbServer};

let server = SmbServer::builder()
    .listen("0.0.0.0:4445".parse()?)
    .share(Share::new("public", LocalFsBackend::new("/srv/public")?).public())
    .build()?;

let config = server.config_handle();

config.add_user("alice", "password").await?;
let home = Share::new("home", LocalFsBackend::new("/srv/home")?)
    .user("alice", Access::ReadWrite);
config
    .add_share(home)
    .await?;
config.revoke_share_user("home", "alice").await?;
config.remove_user("alice").await?;

server.serve().await?;
```

Removing a user revokes that user's active sessions. Removing a share or revoking share access closes matching active tree connections and open handles.


## Feature Status

Implemented:

* SMB2/3 direct-TCP and SMB 3.1.1 over QUIC transports, including ALPN `smb`,
  dialect negotiation, typed wire codecs, and SMB1 multi-protocol negotiate
  bridging.
* Guest and NTLMv2 authentication, authenticated signing, SMB3 encryption,
  compression transforms, transport-security negotiation, and modern SMB 3.1.1
  negotiate contexts.
* Virtual filesystem-backed file serving: share lookup, CREATE/READ/WRITE/CLOSE,
  QUERY_DIRECTORY, QUERY_INFO, SET_INFO, FLUSH, CHANGE_NOTIFY, IOCTL, IPC pipe
  basics, metadata classes, named streams, security descriptors, extended
  attributes, POSIX metadata, and stable file IDs.
* SMB concurrency and resilience behavior: multi-credit validation, 8 MiB WAN
  transfer defaults, compound requests, async pending responses, leases,
  oplocks, cache-break ACK/timeout handling, durable v1/v2 reconnect, replay,
  and lock-sequence replay.
* Interop and performance tooling: `minimal-smb-example`,
  `hnfs-smb-example`, `smbperf`, WAN emulation helpers, CloudSoda/go-smb2,
  Samba `smbclient`, Samba `smbtorture`, macOS `smbutil`, macOS
  `mount_smbfs`, and QUIC smoke/perf coverage.

Future Goals:

* Kerberos authentication and fuller SPNEGO negotiation coverage.
* Persistent durable handles.
* Multi-channel and QUIC multipath.
* Verified native Windows SMB-over-QUIC interop.

## Methodology
Much of this code was generated by iterative testing against [SMBTorture](https://linux.die.net/man/1/smbtorture).
Moving forward, the goal is to achieve similar results without "teaching to the test."
